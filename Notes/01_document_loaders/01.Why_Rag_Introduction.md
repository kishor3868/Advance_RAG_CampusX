### Four Problems in LLMs

1. Knowledge Cutoff - Models frozen in time, can't access recent
information.
2. Hallucinations - Models generate plauble but false information
confidently.
3. No Source Attribution - Can't verify or trace where information came
from.
4. No Private Data Access - Can't access company/proprietary documents.

#### What is Hallucination?

A hallucination happens when an LLM generates:

false information,
made-up facts,
incorrect reasoning,
or imaginary references

while sounding very confident.

The model produces an answer that looks correct linguistically but is actually wrong.

#### Why Do LLMs Hallucinate?

LLMs do not truly understand facts like humans.

They work by:

predicting the next most probable word/token
based on patterns learned during training.

So sometimes they generate:

statistically likely text
instead of
factually correct text. 

#### Main Reasons for Hallucinations

1. Lack of Knowledge / Missing Data
2. Probabilistic Nature of LLMs
3. Ambiguous or Incomplete Prompt
4. Weak Retrieval in RAG Systems
5. Context Window Limitation

### Apart from RAG, other techniques used to add knowledge to LLMs are:

1. Fine-Tuning
    Train the model on domain-specific data so it learns new knowledge permanently.
2. Prompt Engineering / In-Context Learning
    Provide knowledge directly inside the prompt/examples during inference. 
3. Tool Calling / API Integration
    Allow LLM to fetch real-time data from APIs, databases, calculators, or search engines. 

1. Fine-Tuning
What it is

Train the LLM on domain-specific data so knowledge becomes part of the model.

Problem
Expensive and time-consuming
Needs retraining for new data
Cannot easily update frequently changing information
Risk of catastrophic forgetting

Why RAG over it?

RAG can use updated external data instantly without retraining the model.

2. Prompt Engineering / In-Context Learning
What it is

Provide knowledge directly inside the prompt/context.

Problem
Lost in the middle problem(If huge data will be there then llm only take some data from up and down so it lost datat in middle)
Limited by context window
Token cost becomes high
Not scalable for large documents
Important information may get truncated
Why RAG over it?

RAG retrieves only relevant chunks dynamically, making it scalable and cheaper.

3. Tool Calling / API Integration
What it is

LLM calls APIs/tools/databases to fetch live information.

Problem
Requires predefined tools/APIs
Complex orchestration
Not suitable for large unstructured documents
Depends heavily on external systems
Why RAG over it?

RAG is better for searching and reasoning over huge unstructured knowledge bases like PDFs, documents, manuals, etc.
